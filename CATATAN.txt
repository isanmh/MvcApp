### DB Context (Koneksi Database)###
- buat class baru di folder Database => beri nama AppDbContext.cs
- unutk isi AppDbContext.cs seperti berikut :
bisa gunakan snippet vscode :
dbcontext-ef  

- koneksikan di program.cs 
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("SqlConnection")));

- tambahkan connection string di appsettings.json
"SqlConnection": "Server=namaDekstop\\SQLEXPRESS;Database=NamaDatabase;Trusted_Connection=True;MultipleActiveResultSets=true"

"SqlConnection": "Server=DESKTOP-1TSFVMP\\SQLEXPRESS;Database=mvcapp;Trusted_Connection=True;MultipleActiveResultSets=true"

### Migrasi Database ###
- cara installasi Entity Framework Core tools global : 
    dotnet tool install -g dotnet-ef
Eksekusi tools :
    dotnet tool update -g dotnet-ef

- Migrasi :
    php artisan make:migration create_nama_table --create=nama_table

    dotnet ef migrations add InitialCreate

    php artisan migrate:fresh --seed
- masukan ke database :  
    dotnet ef database update

    php artisan migrate

###  alur ###
M - V - C

ORM EF :
_db.Tolist() => ambil semua data
SELECT * FROM Table
?input=backdor

_db.Find(id) => ambil data berdasarkan id
SELECT * FROM Table WHERE Id = id

_db.Add() => menambah data
INSERT INTO Table (kolom1, kolom2) VALUES (nilai1, nilai2)

_db.Update() => mengubah data
_db.Remove() => menghapus data
_db.SaveChanges() => menyimpan perubahan ke database

versi asyncronous :
await _db.TolistAsync()
await _db.FindAsync(id)
await _db.SaveChangesAsync()
await _db.AddAsync()


### cara validasi ###
- [ValidateAntiForgeryToken] di controller
- gunakan data annotation di model
- di controller cek validasi dengan ModelState.IsValid
- di view tampilkan error validasi dengan tag helper asp-validation-for
- di view tambahkan tag helper asp-antiforgery="true" pada form untuk mencegah serangan CSRF (Cross-Site Request Forgery)
